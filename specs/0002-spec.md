# Mirage Fit

Migrage Fit 是一个图片 remix 工具, 它使用 gemini-2.5-flash-image-preview 模型, 通过 prompt remix 多张图片, 生成一张新的图片. 在这个应用中，用户可以上传个人照片，然后和系统提供的各种物品 remix，用户也可以 trigger 生成单独的物品图片。请根据 @specs/0001-nano-banana.md 中的文档，学习如何使用 gemini-2.5-flash-image-preview 模型，构建完善的 prompts（包括生成物品的 prompts），完成图片 remix 的功能。

物品类别有：

- 帽子
- 眼镜
- 鞋子
- 上衣
- 裤子/裙子
- 袜子
- 手套
- 围巾
- 包包
- 饰品
- 其他

在系统第一次使用时，会自动生成一批默认的物品图片，方便用户使用。所有生成或者上传的内容会放在：`~/.mirage-fit/` 目录下：

- 用户上传的照片：`~/.mirage-fit/input/`
- 物品图片：`~/.mirage-fit/items/<item_name>/<xxxx>.jpg` (从0001开始递增)
- 生成的 remix 图片：`~/.mirage-fit/output/<xxxxxx>.jpg` (xxxxxx 是所有参与 remix 的图片的 blake3 hash 值的前6位字符)，这样相同的输入如果已经有结果，则直接返回，避免重复生成。

## 后端

使用 @agent-rust-backend-expert subagent，要求：

1. 使用 axum 0.8 构建 web server
2. 使用 reqwest 0.12 构建对 Gemini API 的调用，所有 gemini API 调用都放在 `src/gemini.rs` 文件中
3. 使用 rust-embed 构建对静态文件的嵌入

## 前端

使用 @agent-ui-engineer subagent, 使用 react/typescript/vite/tailwind/shadcn/d3 构建。主界面参考 @specs/mirage-fit.jpg.

前端代码放在 `ui/` 目录下。
